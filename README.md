# 🏥 Fortis Healthcare: End-to-End Hospital Operations & Patient Care Analytics Pipeline

## 📌 Project Overview

This project demonstrates a complete end-to-end healthcare analytics workflow using Excel, Python, PostgreSQL, and Power BI. The objective was to analyze hospital operations, patient demographics, revenue performance, and admission trends to support data-driven decision-making.

The project simulates a real-world healthcare analytics environment where raw patient records are transformed into actionable business insights through data cleaning, SQL analysis, spreadsheet auditing, and interactive dashboarding.

---

## 🎯 Business Objective

The primary goals of this project were:

- Analyze patient admission patterns
- Monitor hospital revenue performance
- Track department-wise operational efficiency
- Identify high-volume patient segments
- Support executive decision-making through interactive dashboards

---

# 🔄 End-to-End Analytics Workflow

## 1️⃣ Excel – Data Auditing & Business Analysis

### Tasks Performed

- Data validation checks
- Conditional formatting
- Pivot Tables
- Revenue summaries
- Department performance analysis
- VLOOKUP, XLOOKUP, INDEX-MATCH functions

### Outcome

Created audit-ready healthcare reports for quick business analysis.

---

## 2️⃣ Python – Data Cleaning & Preprocessing

### Libraries Used

- Pandas
- NumPy
- Matplotlib
- Seaborn

### Tasks Performed

- Missing value treatment
- Data validation
- Duplicate handling
- Exploratory Data Analysis (EDA)
- Revenue analysis
- Patient demographic analysis

### Output

Generated a clean dataset for downstream analytics and reporting.

---

## 3️⃣ PostgreSQL – Business Intelligence Queries

### SQL Analysis Performed

- Total patient count analysis
- Revenue aggregation
- Department-wise performance analysis
- Emergency admission analysis
- Senior patient segmentation
- Business KPI reporting

### Sample Business Questions

- Which department generates the highest revenue?
